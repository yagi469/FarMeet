# タスク: 管理者ダッシュボードでの削除済み農園表示修正

## 背景
管理者ダッシュボードにおいて、削除済み農園の一覧が表示されない問題が発生していた。
調査の結果、バックエンドのJPAエンティティに付与された `@SQLRestriction("deleted = false")` が原因で、通常のクエリやエンティティのロード時に削除済みデータが除外されていることが判明した。
特に、農園 (`Farm`) とオーナー (`User`) の関係において、オーナーも削除されている場合にプロキシの解決ができずエラーになる可能性があった。

## 目的
管理者ダッシュボードで削除済みの農園一覧を正常に取得・表示できるようにする。

## 対応内容
1. `FarmDto` および `UserDto` クラスの作成。
2. `AdminService` の `getDeletedFarms` メソッドを修正し、エンティティではなく DTO を返すように変更。
3. ネイティブクエリを使用して削除済みデータを取得し、手動で DTO にマッピングすることで JPA の制約を回避。
4. `AdminController` の修正。
