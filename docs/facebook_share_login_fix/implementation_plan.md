# 実装計画: Facebookシェアのログインフロー改善

## 概要
Facebookシェアボタンをクリックした際、未ログインユーザーにエラー画面ではなくログイン画面が表示されるように修正する。

## 現在の問題
- `m.facebook.com/sharer.php` を使用しているため、未ログイン時に「ログインしていません」というエラーページが表示され、ログイン画面に遷移しない。

## 解決策
- シェアURLを `www.facebook.com/sharer/sharer.php` に変更する。
- PC/SP両対応のURLを使用することで、Facebook側の適切なリダイレクト（ログイン画面への誘導）を利用する。

## 変更内容
### フロントエンド
- `components/ShareButtons.tsx`: FacebookのシェアURLを変更

## 検証方法
- 修正後、Vercelにデプロイ
- シークレットウィンドウ等で未ログイン状態を作り、シェアボタンをクリック
- ログイン画面が表示されることを確認
