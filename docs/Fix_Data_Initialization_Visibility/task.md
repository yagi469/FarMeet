# DataInitializerによる初期データが表示されない問題の修正

## 概要
`DataInitializer.java` が実行されているにもかかわらず、アプリケーション上で農園データが表示されない問題を修正する。

## 原因
`Farm` エンティティに `@SQLRestriction("deleted = false")` が設定されているため、論理削除された（`deleted = true`）農園データは通常の JPA クエリ（`farmRepository.count()` など）ではカウントされない。
過去の実行などで農園データが作成され、その後論理削除された状態になっていると、`count()` は 0 を返す。
`DataInitializer` は `count() == 0` の場合に新規作成を行おうとするが、既存のデータ（論理削除済み）と競合する可能性や、あるいは単に「初期化が必要」と判定して作成しようとするが、何らかの理由（重複エラー等）で失敗、または作成されても即座に見えなくなる状況が発生していたと考えられる。
最も可能性が高いのは、**「論理削除されたデータがあるため初期化ロジックが誤動作している（あるいは論理削除データの復元が必要な状態である）」** ことである。

## 対応方針
`DataInitializer` のロジックを修正し、論理削除されたデータも含めた物理的なレコード数を確認するようにする。
もし物理的なデータが存在するが、論理的なデータ（表示可能なデータ）が 0 件の場合、それは「全てのデータが論理削除されている」状態であるため、これらを復元（`deleted = false` に更新）する処理を追加する。
